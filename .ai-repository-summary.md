## Overview

**workflow-orchestration-queue** is a headless agentic orchestration platform that transforms GitHub Issues into autonomous execution orders. The system enables AI agents to operate as background production services, performing multi-step, specification-driven tasks without human intervention.

### Mission Statement

Enable "Zero-Touch Construction" — users open a single specification issue and receive a functional, test-passed branch and PR within minutes.

## Core Components

### 1. Notifier Service (`src/notifier_service.py`)

FastAPI-based webhook receiver ("The Ear").

- **Endpoints:**
  - `POST /webhooks/github` — GitHub webhook handler
  - `GET /health` — Health check
- **Security:** HMAC SHA256 signature verification
- **Responsibilities:** Event validation, triage, queue initialization

### 2. Sentinel Service (`src/orchestrator_sentinel.py`)

Background polling and dispatch service ("The Brain").

- **Polling:** 60-second intervals with jittered exponential backoff
- **Locking:** Assign-then-verify distributed lock pattern
- **Execution:** Shell-bridge dispatch via `devcontainer-opencode.sh`
- **Heartbeat:** Progress comments every 5 minutes

### 3. Queue Interface (`src/queue/github_queue.py`)

Abstract task queue with GitHub Issues implementation.

- **ITaskQueue:** Provider-agnostic interface
- **GitHubQueue:** GitHub Issues + Labels as state machine

### 4. Data Models (`src/models/`)

Shared Pydantic models for type safety and validation.

- **WorkItem:** Canonical task representation
- **TaskType:** PLAN, IMPLEMENT, BUGFIX
- **WorkItemStatus:** Label-mapped states

## State Machine

| Label | State | Description |
|-------|-------|-------------|
| `agent:queued` | Pending | Task validated, awaiting Sentinel |
| `agent:in-progress` | Active | Sentinel claimed, worker running |
| `agent:success` | Terminal | Task completed successfully |
| `agent:error` | Terminal | Task failed |
| `agent:infra-failure` | Sub-state | Container/build failure |
| `agent:stalled-budget` | Sub-state | Cost threshold exceeded |

## Security Considerations

- **HMAC Verification:** All webhooks validated against `WEBHOOK_SECRET`
- **Credential Scrubbing:** Automatic sanitization of tokens/keys in logs
- **Network Isolation:** Worker containers run in isolated Docker network
- **SHA-Pinned Actions:** All GitHub Actions pinned to full commit SHA
- **Non-root Containers:** Services run as unprivileged users
